Royal Braids / Salon Shop - Current Project Features
====================================================

Project Overview
----------------
- Production-ready Supabase + Render + Vercel salon platform for Royal Braids.
- Public customer website for services, gallery, booking, reviews, blog, contact, authentication, and client dashboard.
- Admin console for operational management of bookings, schedule, waitlist, content, reviews, messages, services, security, and admin users.
- Supabase-backed data/auth workflows using Postgres, Auth, RLS, migrations, and approved realtime/browser reads.
- Render backend API for privileged service-role workflows, booking transactions, notification jobs, Cloudinary signing, and admin operations.
- Vercel static frontend deployment for public and admin pages.
- Legacy Firebase production code/config/rules/functions are archived under legacy/firebase-production-archive/.

Public Website Features
-----------------------
- Responsive Royal Braids homepage.
- Branded public splash screen with progress bar and accessible reveal behavior.
- Premium salon branding with hero section, rotating logo, calls to action, and salon contact/location details.
- Dark/light mode and brand theme preset support.
- Mobile navigation menu.
- Smooth section navigation for Home, Gallery, Services, Booking, Dashboard, Reviews, Blog, and Contact.
- SEO metadata, Open Graph metadata, and favicon support.
- Very-small-screen responsive polish for narrow mobile layouts.

Services and Gallery Features
-----------------------------
- JavaScript-driven service catalog.
- Service category tabs and visibility controls.
- Service cards include name, description, price, duration, icon, category, and category label.
- Public gallery with Supabase/Render-backed rendering and fallback content.
- Service-category filters, braid-specific filters, sorting controls, featured rails, before/after images, lightbox behavior, and favorites.
- Cloudinary media integration through Render signing where required.

Booking and Waitlist Features
-----------------------------
- Public appointment booking form.
- Dynamic service and time-slot dropdowns.
- Slot availability from Render/Supabase booking-slot data.
- Transactional slot locking through Render to prevent double booking.
- Booking creation in Supabase bookings and booking_slots workflows.
- Waitlist prompt when the selected slot is no longer available.
- Supabase waitlist_entries data model with queue position and queue size tracking.
- Admin waitlist conversion to confirmed when slot is available.
- Email/WhatsApp notification expectations backed by Render notification outbox.

Client Account Features
-----------------------
- Supabase Auth integration.
- Email/password sign-in and registration.
- Forgot-password and password reset support.
- Auth modal on public website.
- Client dashboard for appointments, reviews, favorites, profile settings, preferences, login history, and security activity.
- Booking reschedule and cancellation flows through Render.

Public Content Features
-----------------------
- Approved public reviews/testimonials.
- Auth-gated review submission with moderation.
- Blog cards with title, excerpt, image, publish date, read time, and read-more URL.
- Contact form backed by Render/Supabase contact message pipeline.
- Notification outbox support for owner/business contact alerts.

Admin Console Features
----------------------
- Dedicated admin page at public/admin.html.
- Supabase Auth admin login.
- Authorized admin access record check through Supabase admin_users.
- Roles: super_admin and admin.
- Permission flags: canManageAdmins, canManageBookings, canManageContent, canManageSecurity.
- Permission-scoped tabs for bookings, waitlist, schedule, gallery, blogs, reviews, messages, services, admins, and security.
- Booking lifecycle actions through Render.
- Waitlist queue and conversion workflows.
- Day/week schedule board.
- Gallery, blog, review, contact, and service/category management.
- Admin user delegation and audit logging.

Security and Automation Features
--------------------------------
- Supabase Auth identity/session model.
- Supabase RLS for browser-facing access boundaries.
- Render middleware for token verification and admin authorization.
- Supabase service-role key stored only on Render.
- Provider secrets stored only in server-side environment variables.
- Login activity, security alerts, account change history, admin security actions, and activity timeline.
- Render-backed incident response actions.
- Protected scheduled-job endpoints using X-Job-Secret.
- Jobs for outbox flushing, booking reminders, expired slot release, and waitlist slot-open notifications.

Technology and Integration Features
-----------------------------------
- HTML, CSS, and vanilla JavaScript frontend.
- Supabase browser adapter.
- Render API adapter.
- Node.js/Express Render backend.
- Supabase Postgres/Auth/RLS/migrations.
- Cloudinary media integration.
- Resend email integration.
- WhatsApp Cloud API integration.
- Font Awesome icons and Google Fonts.

Deployment and QA Features
--------------------------
- Supabase migrations and RLS policies under supabase/.
- Render Blueprint in render.yaml.
- Vercel static frontend deployment from public/.
- Active automated coverage includes JavaScript syntax checks, Vitest unit/architecture tests, backend tests, and Playwright E2E tests.
- Firebase emulator/rules/functions tests are archived and not part of active validation.
- Production readiness checklist in docs/production-signoff.md.
